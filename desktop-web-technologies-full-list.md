## Кроссплатформенные десктоп технологии

### На основе веб-технологий

#### 1. **Electron**
**Что это:** Фреймворк для создания десктоп-приложений используя HTML, CSS, JavaScript. Создан GitHub, работает на движке Chromium и Node.js.

**Платформы:** Windows, macOS, Linux

**Преимущества:**
- Огромное сообщество и экосистема
- Используются веб-технологии (легко найти разработчиков)
- Быстрая разработка
- Доступ к файловой системе и ОС
- Автоматические обновления
- Тысячи готовых библиотек npm

**Недостатки:**
- Большой размер (100+ МБ минимум)
- Высокое потребление памяти (300-500 МБ)
- Медленнее нативных приложений
- Больше энергопотребление

**Кто использует:** Visual Studio Code, Slack, Discord, Spotify, Figma, Notion, Microsoft Teams, Skype, WhatsApp Desktop

---

#### 2. **Tauri**
**Что это:** Современная альтернатива Electron. Вместо встроенного Chromium использует системный браузер (WebView). Написан на Rust.

**Платформы:** Windows, macOS, Linux, iOS, Android (в разработке)

**Преимущества:**
- **Крошечный размер** (3-15 МБ вместо 100+ МБ у Electron)
- **Минимальное потребление памяти** (в 5-10 раз меньше Electron)
- **Безопасность** (Rust + изоляция процессов)
- Быстрая работа
- Меньше энергопотребления
- Использует системный WebView (не тащит весь браузер)
- Frontend на любой технологии (React, Vue, Svelte)
- Нативная интеграция с ОС

**Недостатки:**
- Моложе Electron (меньше проверен временем)
- Меньше готовых решений
- Сложнее настройка (но улучшается)
- Зависимость от системного браузера

**Кто использует:** 1Password 8, Solana Wallet, Twist (мессенджер), многие новые проекты переходят с Electron

**Когда выбирать:** Когда важен размер приложения и производительность. Идеально для стартапов и новых проектов.

---

#### 3. **NW.js (Node-Webkit)**
**Что это:** Похож на Electron, объединяет Node.js и Chromium. Создан Intel.

**Платформы:** Windows, macOS, Linux

**Преимущества:**
- Доступ к DOM напрямую из Node.js
- Можно запускать исходный код (не обязательно компилировать)
- Поддержка Chrome Apps API
- Защита исходного кода (компиляция в бинарник)

**Недостатки:**
- Меньше популярен чем Electron
- Меньше сообщество
- Тоже большой размер

**Кто использует:** Intel XDK, Game Dev Tycoon, Cloudantix, некоторые инструменты разработки

---

#### 4. **Flutter Desktop**
**Что это:** Фреймворк от Google, изначально для мобильных, теперь и для десктопа. Язык Dart.

**Платформы:** Windows, macOS, Linux, iOS, Android, Web

**Преимущества:**
- Один код для 6 платформ
- Красивые интерфейсы (Material Design, Cupertino)
- Собственный движок рендеринга (любой дизайн)
- Высокая производительность (компиляция в нативный код)
- Hot Reload (мгновенная разработка)
- Богатая анимация

**Недостатки:**
- Больший размер приложения (30-50 МБ)
- Язык Dart (нужно учить)
- Десктоп версия еще молодая
- Меньше плагинов для десктопа

**Кто использует:** Ubuntu Store, Google внутренние инструменты, многие мобильные приложения добавляют десктоп

**Когда выбирать:** Когда нужен один код для мобильных и десктопа, красивый UI критичен.

---

#### 5. **Qt Framework**
**Что это:** Мощный C++ фреймворк для кроссплатформенных приложений. Существует с 1995 года.

**Платформы:** Windows, macOS, Linux, Android, iOS, Embedded системы

**Преимущества:**
- **Очень высокая производительность** (нативный C++)
- Нативный вид на каждой платформе
- Огромная функциональность (GUI, сеть, БД, мультимедиа, 3D)
- Проверен временем (25+ лет)
- Отличная документация
- Qt Creator IDE
- Qt Quick (QML для быстрого UI)

**Недостатки:**
- Сложнее для изучения (C++)
- Лицензия (GPL или коммерческая)
- Медленнее разработка чем веб-технологии

**Кто использует:** Autodesk Maya, VLC Media Player, OBS Studio, KDE Desktop, Mercedes-Benz интерфейсы, медицинское оборудование

**Когда выбирать:** Профессиональные приложения, CAD, медиа-редакторы, промышленные системы.

---

#### 6. **Neutralinojs**
**Что это:** Легковесная альтернатива Electron. Использует системный браузер.

**Платформы:** Windows, macOS, Linux

**Преимущества:**
- **Очень маленький размер** (2-3 МБ)
- Минимальное потребление ресурсов
- Простота использования
- Любой frontend фреймворк

**Недостатки:**
- Очень молодой проект
- Ограниченное API
- Маленькое сообщество
- Меньше функций чем Electron/Tauri

**Когда выбирать:** Простые утилиты, инструменты, где размер критичен.

---

#### 7. **Wails**
**Что это:** Go + WebView для десктопа. Go на backend, веб-технологии на frontend.

**Платформы:** Windows, macOS, Linux

**Преимущества:**
- Маленький размер (10-20 МБ)
- Производительность Go
- Простота Go
- Системный WebView
- Шаблоны для популярных фреймворков

**Недостатки:**
- Молодой проект
- Меньше плагинов
- Go экосистема меньше JavaScript

**Кто использует:** Развивающиеся проекты, утилиты для разработчиков

---

#### 8. **Avalonia**
**Что это:** Кроссплатформенный UI фреймворк для .NET (XAML). Похож на WPF.

**Платформы:** Windows, macOS, Linux, iOS, Android, WebAssembly

**Преимущества:**
- Мощь .NET экосистемы
- XAML для UI (знаком WPF/UWP разработчикам)
- Хорошая производительность
- Стилизация и темы
- MVVM паттерн

**Недостатки:**
- Меньше популярен
- Нужно знать C# и .NET
- Сообщество меньше

**Кто использует:** .NET разработчики, корпоративные приложения

---

#### 9. **UNO Platform**
**Что это:** Пишете на C#/XAML, работает везде. Pixel-perfect UI.

**Платформы:** Windows, macOS, Linux, iOS, Android, Web (WebAssembly)

**Преимущества:**
- Один код для всех платформ
- UWP совместимость
- .NET экосистема
- Pixel-perfect UI

**Недостатки:**
- Сложная настройка
- Меньше материалов для обучения

---

#### 10. **Sciter**
**Что это:** Встраиваемый HTML/CSS движок для нативных приложений. Язык TIScript или C++.

**Платформы:** Windows, macOS, Linux

**Преимущества:**
- Маленький размер (5-7 МБ)
- Быстрый рендеринг
- HTML/CSS для UI
- Низкое потребление памяти

**Недостатки:**
- Малоизвестный
- Маленькое сообщество
- Своя версия HTML/CSS (не полный стандарт)

---

### Game Engines (можно использовать для приложений)

#### 11. **Unity**
**Что это:** Игровой движок, но можно делать приложения. C# программирование.

**Платформы:** Windows, macOS, Linux, iOS, Android, WebGL, Консоли

**Преимущества:**
- Мощные 3D/2D возможности
- Огромное сообщество
- Asset Store (магазин готовых компонентов)
- Кроссплатформенность

**Недостатки:**
- Избыточен для простых приложений
- Большой размер
- Лицензия (платная для больших доходов)

**Кто использует:** Интерактивные приложения, визуализации, VR/AR

---

#### 12. **Unreal Engine**
**Что это:** Мощнейший игровой движок. C++ и Blueprints (визуальное программирование).

**Платформы:** Windows, macOS, Linux, iOS, Android, Консоли

**Преимущества:**
- Фотореалистичная графика
- Мощные инструменты
- Blueprints для не-программистов

**Недостатки:**
- Очень большой и сложный
- Долгое обучение
- Избыточен для большинства приложений

**Кто использует:** Архитектурная визуализация, симуляторы, кино-продукция

---

#### 13. **Godot**
**Что это:** Открытый игровой движок. Легкий и простой.

**Платформы:** Windows, macOS, Linux, iOS, Android, Web

**Преимущества:**
- Полностью бесплатный
- Легковесный
- Простота изучения
- Свой язык GDScript (похож на Python)

**Недостатки:**
- Меньше возможностей чем Unity/Unreal
- Меньше сообщество

## Нативные десктоп технологии

### Windows

#### 14. **.NET (WPF - Windows Presentation Foundation)**
**Что это:** Мощная технология от Microsoft для Windows приложений. XAML для UI, C# для логики.

**Платформа:** Windows

**Преимущества:**
- Красивые интерфейсы (векторная графика)
- Data binding
- Анимации и эффекты
- MVVM архитектура
- Интеграция с Windows

**Недостатки:**
- Только Windows
- Медленнее WinForms

**Примеры:** Visual Studio, профессиональные Windows приложения

---

#### 15. **WinForms (Windows Forms)**
**Что это:** Классическая технология для Windows приложений. Простая и быстрая.

**Платформа:** Windows

**Преимущества:**
- Очень простая
- Быстрая разработка
- Drag-and-drop дизайнер
- Много готовых компонентов

**Недостатки:**
- Устаревший вид
- Только Windows
- Ограниченная стилизация

---

#### 16. **UWP (Universal Windows Platform)**
**Что это:** Современная платформа для Windows 10/11 приложений.

**Платформа:** Windows 10/11, Xbox

**Преимущества:**
- Современный дизайн (Fluent Design)
- Песочница (безопасность)
- Windows Store
- Адаптивность

**Недостатки:**
- Только Windows 10+
- Ограниченное API
- Сложнее WPF

---

#### 17. **WinUI 3**
**Что это:** Новейшая UI платформа от Microsoft. Будущее Windows разработки.

**Платформа:** Windows 10/11

**Преимущества:**
- Современный дизайн
- Высокая производительность
- Отделена от ОС (быстрые обновления)
- Fluent Design System

**Недостатки:**
- Очень новая (могут быть баги)
- Меньше материалов

---

#### 18. **Win32 API (C/C++)**
**Что это:** Низкоуровневое API Windows. Прямое общение с ОС.

**Платформа:** Windows

**Преимущества:**
- Максимальный контроль
- Максимальная производительность
- Доступ ко всем функциям Windows

**Недостатки:**
- Очень сложное
- Много кода для простых вещей
- Легко сделать ошибки

---

### macOS

#### 19. **Swift + SwiftUI**
**Что это:** Современный язык Apple и декларативный UI фреймворк.

**Платформа:** macOS, iOS, watchOS, tvOS

**Преимущества:**
- Современный красивый код
- Декларативный UI
- Live preview
- Интеграция с Apple экосистемой
- Высокая производительность

**Недостатки:**
- Только Apple платформы
- Относительно новый

**Примеры:** Новые Mac/iOS приложения

---

#### 20. **Objective-C + AppKit**
**Что это:** Классический способ разработки для macOS.

**Платформа:** macOS

**Преимущества:**
- Полный доступ к macOS API
- Проверен временем
- Много библиотек

**Недостатки:**
- Устаревший синтаксис
- Сложнее Swift
- Только macOS

---

#### 21. **Catalyst**
**Что это:** Технология для портирования iPad приложений на Mac.

**Платформа:** macOS, iOS

**Преимущества:**
- Один код для iPad и Mac
- Быстрая разработка
- SwiftUI/UIKit

**Недостатки:**
- Не всегда нативный вид на Mac
- Ограничения

---

### Linux

#### 22. **GTK (GIMP Toolkit)**
**Что это:** Популярный UI toolkit для Linux. C язык, но есть биндинги для других.

**Платформы:** Linux, Windows, macOS

**Преимущества:**
- Открытый код
- Легковесный
- Широко используется в Linux
- Кроссплатформенный

**Недостатки:**
- C (сложнее)
- На Windows/Mac не нативный вид

**Примеры:** GNOME приложения, GIMP, Inkscape

---

#### 23. **Qt (для Linux)**
**Что это:** Qt также популярен в Linux мире.

**Платформы:** Все

**Примеры:** KDE Desktop, многие Linux приложения

## Веб-фреймворки Frontend

### JavaScript фреймворки

#### 24. **React**
**Что это:** Библиотека от Facebook для создания интерфейсов. Компонентный подход.

**Преимущества:**
- Огромное сообщество (самое большое)
- Богатая экосистема
- Virtual DOM (производительность)
- React Native (мобильные)
- Много вакансий

**Недостатки:**
- Только View слой (нужны доп. библиотеки)
- JSX синтаксис
- Частые изменения подходов

**Кто использует:** Facebook, Instagram, Netflix, Airbnb, Uber

---

#### 25. **Vue.js**
**Что это:** Прогрессивный фреймворк. Простой и гибкий.

**Преимущества:**
- Легкий для изучения
- Отличная документация
- Гибкость (постепенное внедрение)
- Маленький размер
- Vue 3 Composition API

**Недостатки:**
- Меньше крупных компаний используют
- Меньше вакансий чем React

**Кто использует:** Alibaba, Xiaomi, GitLab, Adobe Portfolio

---

#### 26. **Angular**
**Что это:** Полноценный фреймворк от Google. TypeScript.

**Преимущества:**
- Все включено (роутинг, формы, HTTP)
- TypeScript из коробки
- Сильная типизация
- Подходит для крупных приложений
- Google поддержка

**Недостатки:**
- Крутая кривая обучения
- Многословный код
- Больший размер

**Кто использует:** Google, Microsoft, Samsung, Forbes

---

#### 27. **Svelte**
**Что это:** Компилирующий фреймворк (описан выше).

**Преимущества:**
- Минимальный размер
- Высокая производительность
- Простой код
- Встроенная реактивность

**Кто использует:** New York Times, Philips, 1Password

---

#### 28. **SolidJS**
**Что это:** Реактивный фреймворк без Virtual DOM. Очень быстрый.

**Преимущества:**
- Максимальная производительность
- Реактивность на уровне компилятора
- JSX синтаксис (знаком React разработчикам)
- Маленький размер

**Недостатки:**
- Молодой (2021)
- Маленькое сообщество

---

#### 29. **Preact**
**Что это:** Легковесная альтернатива React (3 КБ). Тот же API.

**Преимущества:**
- Крошечный размер
- Совместимость с React
- Быстрый
- Подходит для виджетов

**Недостатки:**
- Меньше функций
- Некоторые React библиотеки не работают

---

#### 30. **Lit (Polymer)**
**Что это:** Легковесная библиотека для Web Components.

**Преимущества:**
- Нативные Web Components
- Маленький размер
- Не нужен build step (опционально)
- Google поддержка

**Недостатки:**
- Меньше популярна

---

#### 31. **Alpine.js**
**Что это:** Минималистичный фреймворк. "Tailwind для JavaScript".

**Преимущества:**
- Крошечный (15 КБ)
- Легкий для изучения
- Не нужен build step
- Декларативный

**Недостатки:**
- Не подходит для сложных приложений

**Примеры:** Простые интерактивные элементы на сайтах

---

#### 32. **Ember.js**
**Что это:** Полноценный фреймворк. Convention over configuration.

**Преимущества:**
- Все включено
- Стабильность
- Ember CLI (мощная)
- Подходит для крупных приложений

**Недостатки:**
- Теряет популярность
- Крутая кривая обучения

**Кто использует:** LinkedIn, Discourse, Apple Music (web)

---

#### 33. **Backbone.js**
**Что это:** Легковесный MVC фреймворк.

**Преимущества:**
- Минимализм
- Гибкость
- Легкий

**Недостатки:**
- Устарел
- Мало используется в новых проектах

---

#### 34. **Knockout.js**
**Что это:** MVVM фреймворк с data binding.

**Преимущества:**
- Простота
- Два-way binding

**Недостатки:**
- Устарел

---

### Meta-фреймворки (Full-stack)

#### 35. **Next.js (React)**
**Что это:** Фреймворк поверх React для production приложений.

**Преимущества:**
- Server-Side Rendering (SSR)
- Static Site Generation (SSG)
- API routes (backend)
- Автоматическая оптимизация
- Отличная производительность
- Vercel поддержка

**Кто использует:** TikTok, Twitch, Hulu, Nike

---

#### 36. **Nuxt.js (Vue)**
**Что это:** Full-stack фреймворк для Vue.

**Преимущества:**
- SSR/SSG
- Роутинг из коробки
- Vuex интеграция
- Модульная архитектура

**Кто использует:** Gitlab, Ecosia

---

#### 37. **SvelteKit (Svelte)**
**Что это:** Full-stack фреймворк для Svelte.

**Преимущества:**
- SSR/SSG
- Роутинг
- Маленький размер
- Высокая производительность

---

#### 38. **Remix**
**Что это:** Full-stack React фреймворк. Фокус на Web Fundamentals.

**Преимущества:**
- Использует нативные Web API
- Отличная производительность
- Прогрессивное улучшение
- Nested routing

---

#### 39. **Astro**
**Что это:** Статический генератор сайтов. Отправляет 0 JavaScript по умолчанию.

**Преимущества:**
- Минимум JavaScript
- Можно использовать любой фреймворк (React, Vue, Svelte)
- Отличная производительность
- Partial Hydration

**Когда использовать:** Контентные сайты, блоги, документация

---

#### 40. **Gatsby**
**Что это:** React фреймворк для статических сайтов. GraphQL.

**Преимущества:**
- Отличное SEO
- Быстрые сайты
- Богатая экосистема плагинов
- GraphQL для данных

**Недостатки:**
- Долгие сборки на больших сайтах

---

#### 41. **Qwik**
**Что это:** Фреймворк с "resumability". Мгновенная загрузка.

**Преимущества:**
- Instant loading (0 JavaScript initially)
- Resumability (не hydration)
- Отличная производительность

**Недостатки:**
- Очень новый (2022)

---

## Веб-фреймворки Backend

### Node.js фреймворки

#### 42. **Express.js**
**Что это:** Минималистичный веб-фреймворк для Node.js.

**Преимущества:**
- Простота
- Гибкость
- Огромное сообщество
- Middleware система
- Легкий

## Выводы и Рекомендации

### Для Десктопа выбирайте:

**Если нужен маленький размер и производительность:**
1. **Tauri** — лучший баланс (3-15 МБ, быстро)
2. **Qt** — профессиональные приложения (C++)
3. **Flutter** — красивый UI, 6 платформ

**Если знаете веб-технологии:**
4. **Electron** — проверен временем, огромное сообщество
5. **Neutralinojs** — легковесная альтернатива
6. **Wails** — если знаете Go

**Если Microsoft экосистема:**
7. **WinUI 3 / WPF** — для Windows
8. **Avalonia / MAUI** — кроссплатформенно

**Если Apple экосистема:**
9. **SwiftUI** — современно и красиво
10. **Catalyst** — портирование с iPad

---

### Для Веба выбирайте:

**Frontend:**
- **React** — если нужна работа и большое сообщество
- **Vue** — если хотите простоту и гибкость
- **Svelte** — если важна производительность и размер
- **Angular** — для enterprise с TypeScript

**Meta-frameworks:**
- **Next.js** — для React SEO и производительности
- **Nuxt** — для Vue
- **SvelteKit** — для Svelte

**Backend:**
- **Node.js (Express/Fastify/Nest)** — если JavaScript
- **Django** — если Python и нужно быстро
- **FastAPI** — если Python и API
- **Laravel** — если PHP
- **Spring Boot** — если Java и enterprise
- **Go (Gin/Fiber)** — если производительность критична

---

### Для Мобильных:

**Кроссплатформенно:**
- **Flutter** — красивый UI, производительность
- **React Native** — если знаете React

**Нативно:**
- **Swift/SwiftUI** — iOS
- **Kotlin/Jetpack Compose** — Android

**Итого:** Выбор зависит от:
- Требований к производительности
- Размера приложения
- Знакомых технологий
- Целевых платформ
- Размера команды
